# Time Limits

## 1. Concept Skeleton
**Definition:** Pre-set session duration to prevent fatigue and impulsive play  
**Purpose:** Reduce tilt, control variance exposure, and improve discipline  
**Prerequisites:** Self-control strategies, session planning

## 2. Comparative Framing
| Limit Type | Example | Benefit | Risk |
|---|---|---|---|
| Fixed time | 2 hours | Predictable | Miss peak games |
| Stop-loss time | End after 2 losses | Protects bankroll | Early exit bias |
| Break cycles | 10 min per hour | Reduces tilt | Flow disruption |

## 3. Examples + Counterexamples
**Simple Example:** Player stops after 2 hours, regardless of result.  
**Failure Case:** Extending sessions after losses amplifies downside.  
**Edge Case:** Short sessions can increase variance for small edges.

## 4. Layer Breakdown
```
Time Control:
├─ Set session duration
├─ Define break intervals
├─ Track time strictly
├─ End on schedule
└─ Review outcomes later
```
